> **नोट:** यह दस्तावेज़ AI द्वारा अनुवादित किया गया है और हो सकता है कि यह 100% सटीक न हो। यदि आपको कोई त्रुटि मिलती है या सुधार के लिए सुझाव हैं, तो कृपया GitHub पर एक समस्या (issue) खोलें।

## CLI संदर्भ

### बुनियादी कमांड

```bash
collet                              # इंटरैक्टिव TUI
collet "fix the bug in main.rs"     # हेडलेस (चलता है और फिर बाहर निकलता है)
collet --continue                   # अंतिम अधूरा सत्र पुनर्स्थापित करें
collet --resume                     # सत्र सूची से चुनें
collet --model glm-5                # मॉडल ओवरराइड करें
```

### सब-कमांड

```bash
collet setup                        # कॉन्फ़िगरेशन फ़ाइल बनाएं (एकाधिक प्रदाता पंजीकृत करें)
collet secure                       # API कुंजी को एन्क्रिप्ट और स्टोर करें
collet unsecure                     # एन्क्रिप्टेड क्रेडेंशियल निकालें
collet status                       # वर्तमान कॉन्फ़िगरेशन जांचें
collet update                       # नवीनतम संस्करण के लिए जाँच करें
collet --version                    # संस्करण प्रदर्शित करें
collet help | -h | --help           # सहायता
```

### प्रदाता प्रबंधन (Provider Management)

```bash
collet provider add                 # एक प्रदाता जोड़ें (अनुशंसित मॉडल चयन UI)
collet provider add openai          # सीधे एक विशिष्ट प्रदाता जोड़ें
collet provider list                # पंजीकृत प्रदाताओं को सूचीबद्ध करें
collet provider use <name>          # सक्रिय प्रदाता बदलें
```

समर्थित प्रदाता: Z.ai, OpenAI, Anthropic, Google Gemini, DeepSeek, Mistral, Groq, Ollama, LM Studio, OpenRouter, Together AI, Azure OpenAI, Amazon Bedrock, Custom

> Ollama और LM Studio स्थानीय सर्वर हैं और इन्हें API कुंजी की आवश्यकता नहीं होती है।

### MCP प्रबंधन

```bash
collet mcp                          # MCP सर्वर कॉन्फ़िगरेशन जांचें
collet mcp add <name> -c <cmd>      # एक MCP सर्वर जोड़ें
collet mcp remove <name>            # एक MCP सर्वर निकालें
```

### CLI फ़्लैग

| फ़्लैग | संक्षिप्त | विवरण |
|------|-------|-------------|
| `--dir <path>` | `-d` | कार्यशील निर्देशिका (प्रोजेक्ट पथ) निर्दिष्ट करें |
| `--continue` | `-c` | अंतिम अधूरे सत्र को स्वतः पुनरारंभ करें |
| `--resume` | `-r` | सत्र सूची पॉपअप से चुनें |
| `--resume <id>` | `-r <id>` | ID द्वारा विशिष्ट सत्र फिर से शुरू करें |
| `--model <name>` | | मॉडल ओवरराइड करें |
| `--yolo` | | ऑटो-कमिट, पुष्टिकरण छोड़ें |
| `--watch` | | फ़ाइल परिवर्तनों पर फिर से चलाएँ |
| `--ext <exts>` | | लक्ष्य एक्सटेंशन देखें (कोमा से अलग) |
| `--debounce <ms>` | | वॉच डिबाउंस (डिफ़ॉल्ट: 2000) |
| `--cwd <path>` | | वॉच निर्देशिका (डिफ़ॉल्ट: वर्तमान स्थान) |
| `--agent <name>` | `-a` | कस्टम एजेंट का उपयोग करें |
| `--fast` | `-f` | तेज़ मोड: कोई साइडबार नहीं, कोई स्वार्म नहीं, कोई सोल प्रतिबिंब नहीं |

### हेडलेस मोड

```bash
collet "fix the login bug"                    # वर्तमान निर्देशिका में चलाएँ
collet "fix the login bug" -d ~/projects/api  # अलग प्रोजेक्ट में चलाएँ
collet "lint this" --yolo                     # ऑटो-कमिट, पुष्टिकरण छोड़ें
collet "lint this" --fast                    # तेज़ मोड: न्यूनतम ओवरहेड
```

### वॉच मोड (फ़ाइल परिवर्तन पहचान)

```bash
# बुनियादी उपयोग
collet "lint this" --watch --ext rs,toml --debounce 500

# केवल विशिष्ट निर्देशिका देखें
collet "run tests" --watch --cwd ./src --ext rs

# कस्टम एजेंट का उपयोग करें
collet "review changes" --watch --agent reviewer --ext ts,tsx
```

फ़ाइलें बदलने पर स्वचालित रूप से पुनरारंभ होता है।

**उपयोग के मामले:**

| परिदृश्य | कमांड |
|----------|---------|
| विशिष्ट सब-डायरेक्टरी देखें | `--watch --cwd ./packages/core` |
| कॉन्फ़िगरेशन फ़ाइल परिवर्तन पहचानें | `--watch --cwd ./config --ext yaml,toml` |
| CI स्क्रिप्ट को ऑटो-फ़िक्स करें | `--watch --cwd .github --ext yml` |
| दस्तावेज़ों को ऑटो-फ़ॉर्मेट करें | `--watch --cwd ./docs --ext md` |
| हल्के मॉडल के साथ तेज़ लिंटिंग | `--watch --agent fast --ext ts` |

### अंतर्निहित एजेंट (Built-in Agents)

कोलेट चार अंतर्निहित एजेंटों के साथ आता है। `Tab` / `Shift+Tab` के साथ उनके बीच स्विच करें।

| एजेंट | भूमिका |
|-------|------|
| **Arbor** (डिफ़ॉल्ट) | पूरी तरह से स्वायत्त ऑर्केस्ट्रेटर — प्रत्येक कार्य के लिए स्वचालित रूप से सिंगल-एजेंट बनाम झुंड (swarm) मोड तय करता है। कोई पॉपअप नहीं, किसी कॉन्फ़िगरेशन की आवश्यकता नहीं। |
| **Architect** | डिज़ाइन और योजना एजेंट। एक संरचित योजना बनाता है, फिर `/proceed` → कोड या योजना सहेजने के लिए `/save-plan` प्रदान करता है। |
| **Code** | कार्यान्वयन कार्यों के लिए डिफ़ॉल्ट कोडिंग एजेंट। |
| **Ask** | बिना टूल एक्सेस के संवादात्मक / प्रश्न-उत्तर एजेंट। |

> **टिप**: जब Arbor सक्रिय होता है, तो आपका कार्य स्वचालित रूप से इष्टतम निष्पादन मोड (Fork / Hive / Flock / single-agent) पर भेज दिया जाता है। अन्य एजेंटों के लिए, आपके चयन को हमेशा प्राथमिक एजेंट के रूप में सम्मानित किया जाता है।

### शेल पासथ्रू (Shell Passthrough)

बिना TUI छोड़े सीधे कार्यशील निर्देशिका में चलाने के लिए किसी भी शेल कमांड के आगे `!` लगाएं:

```
! git log --oneline -5
! cargo test
! ls src/
```

एग्जिट कोड, stdout और stderr चैट में एक कोड ब्लॉक के रूप में प्रदर्शित होते हैं। टर्मिनल स्विच किए बिना त्वरित जांच के लिए उपयोगी।

### @ उल्लेख (Mentions)

| पैटर्न | क्रिया |
|---------|--------|
| `@src/main.rs` | फ़ाइल सामग्री को संदर्भ में संलग्न करें |
| `@src` | फ़ोल्डर ट्री संरचना संलग्न करें (3 स्तर, 200 आइटम) |

| `@glm-5` | निर्दिष्ट मॉडल पर एजेंट स्विच करें |

> **ऑटो-कम्पलीट**: मेंशन पॉपअप खोलने के लिए `@` टाइप करें। एजेंट नाम **फ़ज़ी मैचिंग** का उपयोग करते हैं। फ़ाइल/फ़ोल्डर नाम सब-डायरेक्टरी नेविगेशन के साथ प्रिफ़िक्स मैचिंग का उपयोग करते हैं।

## मल्टी-प्रोवाइडर कॉन्फ़िगरेशन

`/models` कमांड का उपयोग करके रीयल टाइम में कई LLM प्रदाताओं को पंजीकृत और स्विच किया जा सकता है।

### प्रदाता पंजीकरण

```bash
collet provider add          # इंटरैक्टिव विज़ार्ड (अनुशंसित मॉडल सूची से चुनें)
collet provider add openai   # सीधे एक विशिष्ट प्रदाता जोड़ें
```

### config.toml उदाहरण

```toml
[api]
provider = "openai"

# providers = केवल क्रेडेंशियल (मॉडल व्यवहार [[models]] और [[agents.list]] में कॉन्फ़िगर किया गया है)
[[providers]]
name = "openai"
base_url = "https://api.openai.com/v1"
api_key_enc = "..."

name = "ollama"
base_url = "http://localhost:11434/v1"
```

### पदानुक्रमित कॉन्फ़िगरेशन (Global < Model < Agent)

कॉन्फ़िगरेशन प्राथमिकता: `env/[agent]` → `[[models]]` → `[[agents.list]]`

```toml
# प्रति-मॉडल डिफ़ॉल्ट ओवरराइड
[[models]]
name = "gpt-4o"
context_window = 128000
max_output_tokens = 16384
supports_tools = true

[[models]]
name = "qwen2.5-coder"
supports_tools = false
reasoning_effort = "medium"     # low | medium | high

# प्रति-एजेंट ओवरराइड (उच्चतम प्राथमिकता)
[[agents.list]]
name = "code"
model = "gpt-4o"
provider = "openai"              # Tab/Shift+Tab के माध्यम से टॉगल करते समय प्रदाता भी स्विच हो जाता है
temperature = 0.2

[[agents.list]]
name = "architect"
model = "gpt-4o"
provider = "openai"
temperature = 0.7
thinking_budget_tokens = 8192    # reasoning_effort पर प्राथमिकता लेता है

[[agents.list]]
name = "local"
model = "qwen2.5-coder"
provider = "ollama"              # स्थानीय प्रदाता पर स्विच करें
```

`provider` फ़ील्ड को छोड़ने से वर्तमान में सक्रिय प्रदाता बना रहता है और केवल मॉडल बदलता है।

## MCP सर्वर

एजेंट टूल का विस्तार करने के लिए [Model Context Protocol](https://modelcontextprotocol.io) सर्वर कनेक्ट करें।

### सर्वर जोड़ना

```bash
# npm पैकेज से
/mcp add npm:@anthropic/mcp-server-filesystem

# स्थानीय बाइनरी
/mcp add local:/path/to/server --name myserver

# HTTP एंडपॉइंट
/mcp add http:https://api.example.com/mcp

# विश्व स्तर पर जोड़ें (सभी प्रोजेक्ट)
/mcp add npm:@some/tool --global
```

सर्वर स्थिति को साइडबार में जांचा जा सकता है (इंटरैक्टिव टॉगल के लिए `/mcp` कमांड)।

## कोड इंटेलिजेंस

बिना किसी अतिरिक्त सेटअप के स्वचालित रूप से काम करता है।

1. **इंडेक्सिंग** — स्टार्टअप पर, Tree-sitter स्रोत को पार्स करता है और प्रतीकों (फ़ंक्शन, प्रकार, स्थिरांक) को निकालता है।
2. **ऑटो-इंजेक्शन** — उपयोगकर्ता संदेशों का विश्लेषण करता है और स्वचालित रूप से प्रासंगिक फ़ाइलों को संदर्भ में इंजेक्ट करता है।
3. **grep कम करें** — एजेंट को सूचित करता है कि कोडबेस पहले से इंडेक्स किया गया है, जिससे अनावश्यक `grep`/`find` कॉल कम हो जाते हैं।

## मल्टी-एजेंट सहयोग

पदानुक्रम: **Fork** < **Hive** < **Flock**

| मोड | यह कैसे काम करता है | इसके लिए सर्वश्रेष्ठ |
|------|-------------|----------|
| Fork | समन्वयक काम बांटता है → समानांतर निष्पादन → परिणाम मर्ज करें | स्वतंत्र उप-कार्य |
| Hive | एजेंटों के बीच आम सहमति मतदान, समन्वयक निरीक्षण | जटिल निर्णय लेना |
| Flock | रीयल-टाइम एजेंट-टू-एजेंट मैसेजिंग (प्रायोगिक) | कसकर जुड़ा हुआ सहयोग |

## रिमोट गेटवे (Telegram / Slack / Discord)

चैट प्लेटफॉर्म से कोलेट का उपयोग करें। रिमोट कोडिंग कार्य जारी करने के लिए चैनलों को प्रोजेक्ट निर्देशिकाओं में मैप करें।

### CLI प्रबंधन

```bash
collet remote add [platform]   # एडाप्टर जोड़ें (इंटरैक्टिव)
collet remote start            # बैकग्राउंड डेमन के रूप में प्रारंभ करें
collet remote stop             # डेमन रोकें
collet remote status           # वर्तमान स्थिति
```

इंटरैक्टिव रूप से Telegram/Slack/Discord के बीच चयन करने और टोकन, अनुमत उपयोगकर्ता आदि सेट करने के लिए बस `collet remote add` चलाएं।

## Soul.md — एजेंट व्यक्तित्व

प्रत्येक कार्य पूरा होने के बाद, एजेंट सत्र पर विचार करता है और उत्तरोत्तर अपने व्यक्तित्व को `~/.collet/agents/souls/{name}.md` में रिकॉर्ड करता है।

---

## पर्यावरण चर (Environment Variables)

| चर | डिफ़ॉल्ट | विवरण |
|----------|---------|-------------|
| `COLLET_MODEL` | `glm-4.7` | उपयोग करने के लिए मॉडल |
| `COLLET_MAX_TOKENS` | `8192` | अधिकतम प्रतिक्रिया टोकन |
| `COLLET_TOOL_TIMEOUT` | `120` | टूल कॉल टाइमआउट (सेकंड) |
| `COLLET_THEME` | `default` | TUI रंग थीम |
| `COLLET_TELEMETRY` | *(none)* | टेलीमेट्री अक्षम करने के लिए `0` पर सेट करें |

## फ़ाइल संग्रहण पथ

सभी रनटाइम फ़ाइलें OS की परवाह किए बिना XDG-शैली पथों में संग्रहीत की जाती हैं।

```
~/.config/collet/
  └── config.toml              # मुख्य कॉन्फ़िगरेशन

~/.collet/
  ├── logs/collet.log          # लॉग
  ├── SOUL.md                  # ग्लोबल एजेंट व्यक्तित्व
  └── agents/souls/{name}.md   # प्रति-एजेंट व्यक्तित्व
```

## गोपनीयता और टेलीमेट्री

कोलेट उत्पाद को बेहतर बनाने के लिए अनाम उपयोग सांख्यिकी और त्रुटि रिपोर्ट एकत्र करता है।

इसे `COLLET_TELEMETRY=0` सेट करके या `config.toml` में अक्षम किया जा सकता है।

## इवोल्यूशन इंजन

एक स्व-सुधार लूप जहां एजेंट अपनी क्षमताओं को परिष्कृत करता है। यह कार्यों के बैच चलाता है, परिणामों का विश्लेषण करता है, और स्वचालित रूप से प्रॉम्प्ट, कौशल और स्मृति को बदलता है।

```bash
# सरल उपयोग — प्राकृतिक भाषा कार्य + स्वचालित सुधार
collet evolve "fix the login bug"
```

## प्लगइन सिस्टम

प्लगइन्स कोलेट को कस्टम कमांड, कौशल, एजेंट और हुक के साथ विस्तारित करते हैं। वे `~/.collet/plugins/<name>/` में संग्रहीत होते हैं।

```bash
collet plugin install owner/repo
collet plugin list
```
