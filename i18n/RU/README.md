# <center>collet</center>

<center>
**Неутомимый агентный оркестратор кодирования с циклами агентов без потерь.**

Оркестрирует любую LLM, любого CLI-агента через TUI, веб-интерфейс и IDE.

Устраняет проблему «зависания» (stuck), характерную для инструментов на базе Node.js, в самом зародыше, обеспечивая высокоуровневый анализ кода и точное редактирование на безопасном фундаменте Rust.

## Особенности

![Features](../../assets/features.jpg)

| Особенность | Описание |
|---------|-------------|
| **Мультипровайдерность** | Подключение к различным провайдерам LLM с потоковой передачей SSE |
| **Карта репозитория Tree-sitter** | Извлечение символов AST для Rust/Python/TypeScript/Go + ранжирование PageRank |
| **Поиск кода BM25** | Инкрементальная индексация, токенизатор с учетом кода, автоматическая вставка контекста |
| **Инкрементальное хеширование BLAKE3** | Предварительная проверка mtime + хеш контента для повторного парсинга только измененных файлов |
| **Хирургическое редактирование** | Точная замена строк без полной перезаписи файлов |
| **Навыки агента** | 3-уровневая прогрессивная система навыков на основе YAML frontmatter |
| **Субагенты** | Параллельное выполнение подзадач в изолированных окнах контекста |
| **Сохранение сессий** | Восстановление сессии через `--continue` / `--resume` с автосохранением |
| **Автоматическое сжатие** | Сжатие на основе релевантности: дедупликация SimHash + взвешенное сохранение verbatim + структурированное резюме в качестве резервного варианта |
| **Предотвращение зависаний** | Лимиты итераций, автоматические выключатели (circuit breakers), тайм-ауты Tokio |
| **Темы и анимация** | 6 цветовых тем (Dracula, Catppuccin и др.) + анимация размышления со спиннером шрифтом Брайля |
| **Обзор плана архитектора** | Требуется одобрение пользователя для перехода «Архитектор → Код» с поддержкой экспорта файла плана |
| **Параллельный/Командный режим** | Разделение задачи → параллельное выполнение → слияние результатов. Командный режим: межагентное общение в реальном времени + консенсус |
| **LSP/MCP** | 35+ языков, 48+ серверов с автоматическим переключением + интеграция инструментов MCP |
| **Шлюз одобрения инструментов** | 3-уровневое одобрение выполнения инструментов (Yolo/Auto/Manual), переключаемое во время выполнения через `Shift+Tab` |
| **RAG-поиск по документам** | Локальные документы Alcove + внешние сервисы HTTP Bridge. Агент автономно ищет через инструмент `rag_search`; SharedKnowledge автоматически распределяется в режиме Swarm |
| **Soul.md** | Система постоянной личности агента. Агент самостоятельно записывает разделы Identity, Voice, Inner World и Growth, постепенно формируя уникальный характер. Глобальный переключатель или переключатель для конкретного агента. **Один дополнительный вызов API при завершении задачи** (макс. 512 токенов) |
| **Интеграция с IDE** | Нативная поддержка JetBrains + расширение VSCode через сервер ACP |

## Быстрый старт

### Требования

- API-ключ провайдера LLM

### Установка и запуск

#### Скачать бинарный файл

Скачайте последнюю версию со страницы [GitHub Releases](https://github.com/epicsagas/collet/releases).

#### через Homebrew (macOS)

```bash
brew install epicsagas/tap/collet
```

#### через cargo-binstall (быстрее всего — готовые бинарные файлы)

```bash
cargo binstall collet
```

> Сначала требуется установка [`cargo-binstall`](https://github.com/cargo-bins/cargo-binstall):
> `cargo install cargo-binstall`

### Настройка и запуск
```bash
collet setup # или используйте параметр --advanced для более детальных настроек

# TUI
collet

# Headless-режим
collet "hello collet!"
```

## Происхождение и статус

Этот проект начинался как личная площадка для экспериментов, чтобы выжать максимум из моделей GLM. Затем одна функция повлекла за собой другую — новые модели, статьи и парадигмы появлялись быстрее, чем остывали предыдущие.

> «Новые модели, новые статьи, новые парадигмы — быстрее, чем любой лог коммитов может за ними угнаться». — Повторяющийся кошмар.

**Текущее состояние** — проект находится в режиме самообеспечения (dogfooding) через collet, «поедает собственный хвост».

**Стабильность**: основной цикл агента надежен. Параллельное выполнение/Swarm и внешние интеграции все еще находятся в стадии стабилизации.
